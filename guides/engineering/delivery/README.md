# Delivery Guides

Use these guides for review, communication, documentation, diagnostics, and release follow-through.

- [code-review.md](code-review.md) - Review for concrete bugs, regressions, missing safeguards, and test gaps.
- [communication.md](communication.md) - Communicate options, findings, references, and file paths clearly.
- [documentation.md](documentation.md) - Document modules, functions, comments, tests, and component attrs.
- [observability.md](observability.md) - Make important features diagnosable with useful signals.
- [post-release-verification.md](post-release-verification.md) - Verify important changes after deploy.
